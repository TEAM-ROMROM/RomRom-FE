## 🎯 목록 화면 AppFadeSlideIn 스태거 애니메이션 통일 — 전략 문서

관련 이슈: [#849](https://github.com/TEAM-ROMROM/RomRom-FE/issues/849)

---

### 1. 요약

`notification_screen.dart`에만 적용된 `AppFadeSlideIn` + stagger 애니메이션을 미적용 목록 화면 5곳에 일괄 적용한다. 기준 패턴은 `itemBuilder` 내 반환 위젯을 `AppFadeSlideIn(delay: Duration(milliseconds: index * AppMotion.staggerDelayMs), child: ...)` 로 감싸는 것이다.

---

### 2. 배경 및 목적

**문제/필요성**: 앱 전반에 공통 등장 애니메이션 위젯(`AppFadeSlideIn`)이 존재하나 `notification_screen.dart`에서만 stagger 애니메이션이 적용돼 있어 목록 화면 간 시각적 일관성이 없음.

**목표**: 모든 목록 화면에 동일한 등장 연출(opacity 0→1, translateY stagger) 적용.

**범위 포함**:
- `my_register_item_screen.dart` — SliverChildBuilderDelegate (짝수=아이템, 홀수=구분선)
- `chat_tab_screen.dart` — SliverChildBuilderDelegate
- `my_like_list_screen.dart` — ListView.separated
- `block_management_screen.dart` — ListView.separated
- `request_management_tab_screen.dart` — List.generate in Column (받은 요청 + 보낸 요청)

**범위 제외**:
- `request_management_tab_screen.dart`의 `PageView.builder` (카드 스와이퍼, 목록 아님)
- 로딩 인디케이터 아이템 (스켈레톤/로딩 위젯은 애니메이션 불필요)

---

### 3. 요구사항

**필수 (P0)**:
- 5개 파일의 itemBuilder/delegate에 `AppFadeSlideIn` + stagger delay 적용
- stagger delay: `Duration(milliseconds: index * AppMotion.staggerDelayMs)`
- 기준 화면(`notification_screen.dart`)과 동일한 패턴 사용

**중요 (P1)**:
- `app_fade_slide_in.dart`, `app_motion.dart` import 추가 (누락 파일만)
- 로딩 인디케이터 아이템은 애니메이션 제외 (`my_like_list_screen` 마지막 index)

**선택 (P2)**:
- 없음 (기존 패턴 그대로 적용)

---

### 4. 선택한 접근 방식

**방식**: 직접 파일 수정 (itemBuilder 반환 위젯을 AppFadeSlideIn으로 감싸기)

**이유**: `AppFadeSlideIn`은 이미 앱 전역 공통 위젯으로 존재. 추가 추상화 불필요. 각 화면의 itemBuilder 패턴만 수정하면 됨.

**대안**: 별도 유틸 헬퍼 함수로 래핑 → 오버엔지니어링, 현재 스코프에서 불필요.

---

### 5. 파일별 작업 상세

| 파일 | 리스트 패턴 | delay 계산 | import 필요 |
|------|------------|-----------|------------|
| `my_register_item_screen.dart` | `SliverChildBuilderDelegate` (짝수=아이템) | `(index ~/ 2) * staggerDelayMs` | `app_fade_slide_in.dart`만 (app_motion 이미 있음) |
| `chat_tab_screen.dart` | `SliverChildBuilderDelegate` | `index * staggerDelayMs` | `app_fade_slide_in.dart`, `app_motion.dart` 모두 |
| `my_like_list_screen.dart` | `ListView.separated` | `index * staggerDelayMs` | `app_fade_slide_in.dart`, `app_motion.dart` 모두 |
| `block_management_screen.dart` | `ListView.separated` | `index * staggerDelayMs` | `app_fade_slide_in.dart`, `app_motion.dart` 모두 |
| `request_management_tab_screen.dart` | `List.generate` × 2 | `index * staggerDelayMs` | `app_fade_slide_in.dart`만 (app_motion 이미 있음) |

#### 구현 패턴 (기준: notification_screen)

```dart
// ✅ 적용 패턴
itemBuilder: (context, index) {
  return AppFadeSlideIn(
    delay: Duration(milliseconds: index * AppMotion.staggerDelayMs),
    child: <기존_아이템_위젯>,
  );
}
```

#### my_register_item_screen 특이사항

`SliverChildBuilderDelegate`는 짝수 인덱스만 아이템(홀수는 Divider 구분선):

```dart
delegate: SliverChildBuilderDelegate((context, index) {
  if (index.isOdd) {
    return Padding(...); // 구분선 — 애니메이션 불필요
  }
  final item = displayItems[index ~/ 2];
  return AppFadeSlideIn(
    delay: Duration(milliseconds: (index ~/ 2) * AppMotion.staggerDelayMs),
    child: Padding(
      padding: EdgeInsets.symmetric(horizontal: 24.0.w),
      child: _buildItemTile(item, index ~/ 2),
    ),
  );
}, childCount: itemCountWithSeparators),
```

#### my_like_list_screen 특이사항

마지막 인덱스(`index == _items.length`)는 로딩 인디케이터이므로 애니메이션 제외:

```dart
itemBuilder: (context, index) {
  if (_hasMoreItems && index == _items.length) {
    return const Padding(...); // 로딩 인디케이터 — 그대로 유지
  }
  final item = _items[index];
  return AppFadeSlideIn(
    delay: Duration(milliseconds: index * AppMotion.staggerDelayMs),
    child: GestureDetector(...), // 기존 아이템 위젯
  );
}
```

#### request_management_tab_screen 특이사항

`List.generate` 패턴 — `_buildReceivedRequestsList`와 `_buildSentListBody` 각각 적용:

```dart
// 받은 요청
children: List.generate(filteredRequests.length, (index) {
  return AppFadeSlideIn(
    delay: Duration(milliseconds: index * AppMotion.staggerDelayMs),
    child: Column(children: [RequestListItemCardWidget(...)]),
  );
})

// 보낸 요청
children: List.generate(_sentRequests.length, (index) {
  return AppFadeSlideIn(
    delay: Duration(milliseconds: index * AppMotion.staggerDelayMs),
    child: Padding(padding: ..., child: SentRequestItemCard(...)),
  );
})
```

### 6. 고려사항 및 위험요소

**기술적 위험**:
- `SliverChildBuilderDelegate`에서 짝수/홀수 구분 시 delay가 `index ~/ 2` 기준임을 유지해야 함 → 구분선에 `AppFadeSlideIn` 감싸지 않도록 주의

**비즈니스 위험**:
- 애니메이션이 너무 많은 경우 UX 피로감 → 기존 `staggerDelayMs=30ms` 값 변경 없이 그대로 사용
