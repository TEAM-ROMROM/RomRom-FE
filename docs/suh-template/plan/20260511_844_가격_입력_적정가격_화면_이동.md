# 🎯 가격 입력 시 적정가격 입력 화면으로 이동 전략 문서

> 이슈: [#844](https://github.com/TEAM-ROMROM/RomRom-FE/issues/844)

---

## 1. 요약

현재 `register_input_form.dart`의 가격 입력은 인라인(AI 안내 배너 + 토글 스위치 + TextField가 한 화면에 혼재)으로 구성되어 있다. 가격 TextField를 탭하면 **전용 가격 입력 스크린(`ItemPriceInputScreen`)으로 Push 이동**하여 AI 추천 가격 토글 + 직접 입력을 처리하고, 완료 시 가격과 `useAiPrice` 값만 등록 폼으로 반환하는 방식으로 개선한다.

---

## 2. 배경 및 목적

**문제/필요성**:
- AI 안내 배너, AI 토글 Row, 가격 TextField 3개 UI 요소가 연달아 배치되어 화면이 복잡
- AI 토글 활성화 → 로딩 → 결과 반영 흐름이 같은 스크롤 화면에서 일어나 직관성 부족

**목표**: 가격 입력을 전용 스크린으로 분리하여 집중적인 가격 입력 UX 제공

**범위**:
- 포함: 가격 입력 스크린 신규 구현, `register_input_form.dart` 가격 섹션 수정
- 제외: 백엔드 API 변경, AI 가격 예측 로직 변경

---

## 3. 요구사항

**필수 (P0)**:
- 가격 TextField 탭 시 `ItemPriceInputScreen`으로 이동
- 스크린 내부: AI 추천 가격 토글 카드 + 가격 입력 필드 + 완료 버튼
- 완료 버튼 탭 시 가격 + `useAiPrice` 값이 등록 폼에 반영
- 기존 `isFormValid`, `isAiPredictedPrice` 서버 전송 로직 유지

**중요 (P1)**:
- `canUseAiPrice` 조건(제목·설명·물건상태 입력 여부) 스크린으로 전달하여 동일하게 동작
- AI 로딩 중 완료 버튼 비활성화
- AI 예측 실패 시 스낵바 표시 후 직접 입력 모드로 전환
- 진입 시 가격 입력 TextField에 숫자 키보드 자동 포커스

**선택 (P2)**:
- 수정 모드에서 기존 가격 + `useAiPrice` 초기값 유지

## 4. 선택한 접근 방식

**방식**: 전체 화면 스크린 Push (`context.navigateTo`)

**이유**:
- 위치 선택(`ItemRegisterLocationScreen`)과 동일한 패턴 — 등록 폼에서 세부 입력 화면으로 이동 후 결과 반환
- 숫자 키보드 자동 포커스 시 전체 화면이 안전 (BottomSheet는 키보드 + 시트 높이 충돌 가능)
- 디자인 목업이 전체 화면 구조로 확정됨

**대안 — BottomSheet**:
- 선택하지 않은 이유: 키보드 올라올 때 레이아웃 overflow 처리가 복잡하고, 디자인 목업이 전체 화면 구조임

## 6. 주요 결정사항

| 결정 | 선택 | 이유 |
|------|------|------|
| UI 방식 | 전체 화면 스크린 Push | 디자인 목업 확정, 키보드 충돌 방지 |
| 파일 위치 | `lib/screens/item_price_input_screen.dart` | `ItemRegisterLocationScreen`과 동일 위치 |
| 상태 위치 | `_measureAiPrice` 로직을 스크린 내부로 이전 | 스크린이 자체 완결된 가격 입력 단위 |
| 반환값 | `({int price, bool useAiPrice})` record | 두 값 모두 등록 폼에서 필요 |
| `useAiPrice` 원본 위치 | `register_input_form.dart`에 유지 | 서버 전송 시 `isAiPredictedPrice` 필드에 사용 |
| 등록 폼 가격 표시 | readOnly TextField (현재와 동일) | 탭 시 스크린으로 이동 |
| 네비게이션 타입 | `NavigationTypes.slideUp` | 하단에서 올라오는 전환이 가격 입력 진입에 자연스러움 |

### Task 1: `ItemPriceInputScreen` 스크린 신규 구현

**파일**: `lib/screens/item_price_input_screen.dart`

```
생성자 파라미터:
  - int initialPrice               // 현재 설정된 가격
  - bool initialUseAiPrice         // AI 추천 가격 사용 여부
  - bool canUseAiPrice             // AI 사용 가능 조건 충족 여부
  - String itemName                // AI 예측 요청용
  - String itemDescription         // AI 예측 요청용
  - String? itemCondition          // AI 예측 요청용

반환 (Navigator.pop):
  - ({int price, bool useAiPrice})  // 완료 버튼 탭 시
  - null                            // 뒤로가기 시

내부 구성:
  1. CommonAppBar (title: '적정 가격', 기본 뒤로가기)
  2. AI 추천 가격 카드
     - 그라디언트 테두리 Container (AppColors.aiGradient)
     - 좌측: "✦AI 추천 가격" GradientText + 설명 텍스트
     - 우측: AnimatedToggleSwitch (기존 로직 이전)
     - canUseAiPrice false 시 탭 → 스낵바 안내
  3. 가격 입력 TextField
     - prefixText: '₩'
     - keyboardType: TextInputType.number
     - inputFormatters: [PriceCommaFormatter()]
     - autofocus: true (진입 시 자동 포커스)
     - readOnly: useAiPrice || _isAiPriceLoading
     - suffixIcon: 로딩 중 CommonLoadingIndicator
  4. Spacer
  5. CompletionButton
     - buttonText: '완료'
     - isEnabled: price > 0
     - isLoading: _isAiPriceLoading
     - enabledOnPressed: Navigator.pop(context, (price: ..., useAiPrice: ...))
```

### Task 2: `register_input_form.dart` 가격 섹션 수정

**제거 대상**:
- AI 안내 배너 Container (line 717~755)
- AI 가격 추천 스위치 Row (line 757~824)
- `_measureAiPrice()` 메서드 (line 155~198)
- `_isAiPriceLoading` 상태 변수 (line 62)
- `animated_toggle_switch` import

**유지**:
- `useAiPrice`, `priceController` 상태 (서버 전송용)
- `canUseAiPrice` getter (스크린에 전달용)

**수정**:
- 가격 TextField: `readOnly: true`, `onTap`에서 `ItemPriceInputScreen`으로 이동
- 반환값으로 `priceController.value`, `useAiPrice` setState 업데이트

```dart
// 가격 필드 onTap 처리
onTap: () async {
  final result = await context.navigateTo<({int price, bool useAiPrice})>(
    type: NavigationTypes.slideUp,
    screen: ItemPriceInputScreen(
      initialPrice: int.tryParse(priceController.text.replaceAll(',', '')) ?? 0,
      initialUseAiPrice: useAiPrice,
      canUseAiPrice: canUseAiPrice,
      itemName: titleController.text,
      itemDescription: descriptionController.text,
      itemCondition: selectedItemConditionTypes.isNotEmpty
          ? selectedItemConditionTypes.first.serverName
          : null,
    ),
  );
  if (result != null) {
    setState(() {
      useAiPrice = result.useAiPrice;
      priceController.value = const PriceCommaFormatter().formatEditUpdate(
        const TextEditingValue(),
        TextEditingValue(text: result.price.toString()),
      );
    });
  }
},
```

## 9. 고려사항 및 위험요소

**기술적 위험**:
- `ItemApi().pricePredict()` 호출 중 스크린 pop → `mounted` 체크 필수
- AI 토글 ON 상태에서 뒤로가기 → `null` 반환이므로 등록 폼 가격 변경 없음 (의도된 동작)

**UX 위험**:
- `autofocus: true` + AI 토글 ON → 키보드가 올라오지만 readOnly → 어색함. AI 토글 ON 시 키보드 dismiss 처리 필요 (`FocusManager.instance.primaryFocus?.unfocus()`)
