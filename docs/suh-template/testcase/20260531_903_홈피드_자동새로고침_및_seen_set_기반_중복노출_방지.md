## 🧪 테스트 케이스: 홈 피드 자동 새로고침 + seen-set 기반 중복 노출 방지
**이슈**: #903 (이슈 #848 대체) **PR**: #904 **담당자**: @Cassiiopeia

---

### ✅ TC-01. Cold Start — 풀스크린 스켈레톤 동작

- [ ] 앱 완전 종료 후 재실행 → 홈 탭 진입 시 **풀스크린 스켈레톤(HomeFeedSkeleton)** 노출 확인
- [ ] 스켈레톤 → 실제 피드로 자연스럽게 전환 확인
- [ ] cold start 시점에는 상단 얇은 progress 바가 노출되지 **않는지** 확인

---

### ✅ TC-02. 탭 재진입 자동 새로고침 — 기본 동작

- [ ] 앱 실행 후 홈 탭에서 카드 3~5개 스크롤 다운
- [ ] 채팅 탭 클릭 → 다시 홈 탭 클릭
- [ ] 자동으로 **page 0으로 점프** (맨 위 카드로 이동) 확인
- [ ] 상단에 얇은 노란색 progress 바(2px, primaryYellow) 잠깐 노출 확인
- [ ] 새로고침 완료 후 progress 바 자동 사라짐 확인
- [ ] 풀스크린 스켈레톤은 **나오지 않아야** 함 (cold start와 자동 새로고침 구분)

---

### ✅ TC-03. 30초 Throttle 동작

- [ ] 홈 탭에서 채팅 탭 → 홈 탭 즉시 복귀 → 새로고침 발생
- [ ] **30초 이내**에 다시 채팅 탭 → 홈 탭 복귀 → 새로고침 **발생 안 함**(상단 progress 바 안 보임, 네트워크 호출 없음)
- [ ] **30초 경과 후** 채팅 탭 → 홈 탭 복귀 → 새로고침 정상 발생
- [ ] (개발자 도구) Charles/네트워크 인스펙터로 30초 내 GET items 호출 횟수가 정확히 1회인지 확인

---

### ✅ TC-04. 포그라운드 복귀 자동 새로고침

- [ ] 홈 탭 상태에서 홈 버튼으로 앱 백그라운드 전환
- [ ] 1분 이상 대기 후 앱 재진입 → 자동 새로고침 발생 확인 (상단 progress 바 + page 0 점프)
- [ ] 채팅 탭 상태에서 백그라운드 전환 후 복귀 → 채팅 탭 그대로 유지, 홈 새로고침 발생 **안 함**
- [ ] 채팅 탭에서 백그라운드 → 복귀 → 그 후 홈 탭 클릭하면 새로고침 발생 확인 (TC-02 경로)

---

### ✅ TC-05. seen-set 재정렬 (중복 노출 방지)

- [ ] 홈 탭에서 카드 5개 스크롤 다운 (5개 모두 본 상태)
- [ ] 30초 대기 후 다른 탭 → 홈 탭으로 새로고침 트리거
- [ ] 새로고침 후 **방금 본 5개 카드가 응답 끝으로 밀려있는지** 확인 (위에서 보이는 카드는 새로운 것 또는 아직 안 본 것)
- [ ] (개발 모드) `[HomeFeed] refresh start` 로그 확인

---

### ✅ TC-06. seen-set LRU 100개 제한

- [ ] 홈 탭에서 100개 이상 카드 스크롤 (loadMore로 추가 로드 포함)
- [ ] 다른 탭 → 홈 탭으로 새로고침 트리거
- [ ] **가장 먼저 본 카드들은 더 이상 응답 끝으로 가지 않음** (LRU 제거되어 unseen 취급)
- [ ] 가장 최근에 본 카드들은 계속 응답 끝으로 재정렬됨

---

### ✅ TC-07. 무한 순환 + seen-set 클리어

- [ ] 홈 탭에서 끝까지 스크롤 (loadMore가 빈 페이지 반환 시점까지)
- [ ] 빈 페이지 응답 시 자동으로 **page 0으로 되감기** + seen-set 클리어 확인
- [ ] 되감긴 후에는 초기 순서대로 카드가 다시 나타남 확인 (재정렬 적용 안 됨)

---

### ✅ TC-08. 자동 새로고침 silent fail (네트워크 끊김)

- [ ] 홈 탭에서 데이터 정상 로딩 확인
- [ ] **비행기 모드 ON** 또는 네트워크 차단
- [ ] 다른 탭 → 홈 탭 (30초 후) → 자동 새로고침 시도 → API 실패
- [ ] **SnackBar 안 뜨고**, **기존 피드 그대로 유지** 확인
- [ ] 상단 progress 바는 잠깐 나왔다가 사라짐 확인
- [ ] 비행기 모드 OFF 후 다시 트리거 → 정상 새로고침 확인

---

### ✅ TC-09. loadMore (사용자 가시 스크롤 로딩)

- [ ] 홈 탭에서 끝까지 스크롤 → 하단에 로딩 인디케이터 (`CommonLoadingIndicator`) 노출 확인
- [ ] 추가 페이지 로드 완료 후 인디케이터 사라지고 새 카드 append 확인
- [ ] loadMore 실패 시(네트워크 끊김 등) **SnackBar로 에러 노출** 확인 (자동 새로고침과 다름 — 사용자 가시 액션)

---

### ✅ TC-10. 광고 슬롯 동작 (회귀 검증)

- [ ] 홈 탭 진입 후 5번째 카드까지 광고 없음 확인
- [ ] 5번째 이후 8~11개 간격으로 광고 슬롯 노출 확인
- [ ] 자동 새로고침 후 광고 슬롯 인덱스도 처음부터 다시 계산되는지 확인 (첫 5개 광고 없음 → 그 뒤 광고 노출 패턴 반복)
- [ ] 광고 슬롯 표시 중에는 상단 알림 아이콘/메뉴 버튼 숨김 확인

---

### ✅ TC-11. 블러 상태 (등록 가능 카드 0개)

- [ ] 등록된 물건이 0개인 계정으로 로그인 → 홈 피드 블러 처리 확인
- [ ] 블러 상태에서 다른 탭 → 홈 탭 복귀 → 자동 새로고침 트리거되지만 화면 변화 없음 (블러 그대로)
- [ ] 등록하기 버튼 정상 동작 확인

---

### ✅ TC-12. 코치마크 + 새로고침 상호작용

- [ ] 첫 물건 등록 후 홈 탭 자동 진입 + 코치마크 노출 확인
- [ ] 코치마크 닫은 후 다른 탭 → 홈 탭 → 자동 새로고침 정상 동작 (코치마크 다시 안 뜸)

---

### ✅ TC-13. 카드 드롭(거래 요청) 회귀 검증

- [ ] 홈 피드 현재 카드에 내 카드를 드래그&드롭 → 거래 요청 화면 정상 이동
- [ ] 이미 존재하는 거래는 "이미 교환 요청이 존재합니다" SnackBar 확인
- [ ] 광고 슬롯이 보이는 동안에는 카드 덱 드래그 비활성 (`dragEnabled: false`)

---

### ✅ TC-14. 신고 메뉴 / 알림 아이콘 (회귀 검증)

- [ ] 현재 보고 있는 카드에서 우상단 신고 메뉴 → 신고 화면 정상 이동 → 신고 접수 후 모달
- [ ] 알림 아이콘 탭 → 알림 화면 이동 → 복귀 시 미확인 알림 갱신 확인

---

### ✅ TC-15. iPad / 대형기기 회귀 검증

- [ ] iPad에서 홈 탭 진입 → 카드 정상 표시
- [ ] 탭 재진입 시 상단 progress 바 위치가 안전영역(SafeArea top) 바로 아래에 정확히 표시되는지 확인
- [ ] 광고 슬롯, 카드 덱, 하단 등록 버튼 모두 기존 동작 그대로 유지 확인
