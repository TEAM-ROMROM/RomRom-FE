## 🧪 테스트 케이스

### TC-01: 국내 좌표 선택 — 정상 동작 회귀 확인
**목적**: 기존 국내 위치 등록 플로우가 회귀 없이 동작하는지 확인
**절차**:
1. 물품 등록 → 위치 설정 진입
2. 지도가 현재 위치(국내) 또는 서울시청 폴백으로 표시되는지 확인
3. 지도를 살짝 이동하여 다른 국내 위치에 핀
4. 카메라 정지 후 약 1초 대기

**기대 결과**:
- 안내 배너 미노출
- '선택 완료' 버튼 활성
- 버튼 탭 시 정상적으로 LocationAddress가 콜백되고 화면 pop

---

### TC-02: 해외 좌표 첫 선택 — 버튼 비활성 + 안내 노출
**목적**: 해외 좌표에서 버튼이 비활성화되고 안내가 표시되는지 확인
**절차**:
1. 위치 설정 진입
2. 지도를 도쿄(35.6762, 139.6503) 부근으로 드래그
3. 카메라 정지 후 약 1초 대기

**기대 결과**:
- '국내 위치만 등록할 수 있어요' 안내 배너가 버튼 위에 노출
- '선택 완료' 버튼 비활성 (탭해도 반응 없음)

---

### TC-03: 국내 → 해외 이동 — stale 주소 미등록 (핵심 회귀 방지)
**목적**: 기존 stale state 버그가 재발하지 않는지 확인
**절차**:
1. 국내 좌표에서 버튼 활성 상태 확인 (TC-01 상태)
2. 그 상태에서 지도를 해외(도쿄)로 드래그
3. 카메라 정지 후 약 1초 대기

**기대 결과**:
- 안내 배너 즉시 노출
- 버튼이 **비활성으로 전환** (이전 국내 주소가 남아 활성 상태로 머무르지 않아야 함)
- (회귀 시 증상: 버튼 활성 + 탭하면 직전 국내 주소가 등록됨)

---

### TC-04: 해외 → 국내 복귀 — 안내 사라지고 버튼 재활성
**목적**: 해외에서 국내로 돌아왔을 때 정상 복구되는지 확인
**절차**:
1. TC-02 상태(해외, 안내 노출, 버튼 비활성)에서 시작
2. 지도를 다시 국내(서울 등)로 드래그
3. 카메라 정지 후 약 1초 대기

**기대 결과**:
- 안내 배너 사라짐
- 버튼 활성
- 버튼 탭 시 새 국내 주소가 정상 등록

---

### TC-05: 위치 권한 거부 폴백 — 서울시청 좌표 회귀
**목적**: 권한 거부 시 폴백 동작이 깨지지 않았는지 확인
**절차**:
1. 설정에서 앱 위치 권한 거부
2. 위치 설정 진입

**기대 결과**:
- 지도가 서울시청 좌표로 표시
- 안내 배너 미노출
- 버튼 활성 상태

---

### TC-06: `initialLocation` 재진입 — 이전 선택 복원
**목적**: 이전에 선택한 위치로 재진입 시 정상 복원되는지 확인
**절차**:
1. 국내 위치를 한 번 선택해 LocationAddress 저장
2. 위치 화면을 나갔다가 다시 진입 (`initialLocation` 전달)

**기대 결과**:
- 저장된 위치로 카메라 이동
- 안내 배너 미노출
- 버튼 활성 상태

---

### TC-07: 국경 인접 좌표 — 경계 케이스
**목적**: 네이버 API가 응답하지 못하는 경계 좌표에서 UI가 일관되게 동작
**절차**:
1. 독도/대마도/접경 해상 등 응답이 빈 영역으로 핀 이동

**기대 결과**:
- API가 빈 결과를 주면 해외와 동일하게 안내 노출 + 버튼 비활성
- 앱 크래시/무한 로딩 없음

---

### TC-08: 카메라 idle 디바운스 — 빠른 드래그
**목적**: 카메라를 빠르게 여러 번 움직여도 마지막 위치 기준으로만 상태가 결정되는지 확인
**절차**:
1. 국내 → 해외 → 국내 → 해외 순으로 빠르게 드래그
2. 마지막 정지 위치가 해외인지 국내인지 확인

**기대 결과**:
- 최종 핀 위치 기준으로 안내/버튼 상태가 일치
- 중간 상태가 끼어들어 잘못 활성/비활성되지 않음
