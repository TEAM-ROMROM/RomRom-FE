## 🧪 테스트 케이스: 채팅방 목록 마지막 메시지 최신순 정렬

**이슈**: #884
**담당자**: @Cassiiopeia
**PR**: #905
**도메인**: 채팅 (Flutter)

> 핵심: 채팅방 목록은 항상 **마지막 메시지 시각(`lastMessageTime`) 내림차순**(최신이 맨 위)으로 표시되어야 하며, 초기 로드·새로고침·페이징·WebSocket 수신·재연결 **모든 경로**에서 정렬이 일관되게 유지되어야 한다.

---

### ✅ 1. 기본 정렬 동작 확인

- [ ] **초기 로드 정렬**: 채팅방 2개 이상 존재 상태에서 채팅 탭 진입 → 마지막 메시지가 가장 최근인 방이 맨 위에 표시
- [ ] **내림차순 일관성**: 위에서 아래로 내려갈수록 마지막 메시지 시각이 과거인지 확인
- [ ] **새 메시지 수신 시 이동**: 가장 아래(오래된) 채팅방에 새 메시지 수신 → 해당 방이 즉시 목록 최상단으로 이동
- [ ] **내가 보낸 메시지**: 내가 메시지 전송 → 해당 방이 최상단으로 이동하고 unreadCount는 증가하지 않음

### ✅ 2. 갱신 경로별 정렬 유지 (회귀 핵심 — #884가 깨졌던 지점)

- [ ] **앱 재진입**: 앱 완전 종료 후 재실행 → 채팅 탭 진입 시 최신순 정렬 유지
- [ ] **당겨서 새로고침 (Pull-to-Refresh)**: 목록을 아래로 당겨 새로고침 → 정렬 유지
- [ ] **무한 스크롤(페이징)**: 채팅방이 많아 2페이지 이상 로드되는 경우, 스크롤로 다음 페이지 로드 후에도 전체 목록이 최신순 유지 (페이지 경계에서 순서 역전 없음)
- [ ] **WebSocket 재연결**: 네트워크 끊김 → 재연결 후 재조회 시 정렬 유지 (재연결 중 도착한 메시지 반영 포함)
- [ ] **백그라운드 → 포그라운드**: 앱을 백그라운드로 보냈다가 복귀 → 그 사이 받은 메시지가 반영되며 정렬 유지

### ⚠️ 3. 엣지 케이스

- [ ] **메시지 없는 방 (lastMessageTime null)**: 메시지를 한 번도 주고받지 않은 채팅방은 목록 **맨 뒤**에 배치 (BE가 createdDate로 채워주므로 실제 null은 드물지만, null인 경우 후순위 확인)
- [ ] **메시지 있는 방 + 없는 방 혼재**: 메시지 있는 방들이 위, 없는 방이 아래로 분리되는지
- [ ] **동일 시각 메시지**: 여러 방의 마지막 메시지 시각이 (거의) 동일할 때 앱이 깨지거나 깜빡임 없이 표시
- [ ] **채팅방 1개만 존재**: 정렬 로직이 단일 항목에서도 정상 (크래시 없음)
- [ ] **채팅방 0개 (빈 목록)**: 빈 상태 화면 정상 노출, 에러 없음
- [ ] **차단된 방 제외**: 차단(blocked) 채팅방은 목록에서 제외된 상태로 정렬됨
- [ ] **방 나가기/삭제 후**: 채팅방 나간 직후 목록에서 즉시 제거되고 나머지 정렬 유지
- [ ] **중복 메시지 수신**: 같은 메시지가 중복 도착해도 목록 순서/중복 표시 이상 없음
