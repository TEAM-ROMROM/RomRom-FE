### 💡 Minor (개선 권장)

**`app_review_service.dart:77`** — 쿨타임 비교 연산자
- 현재: `DateTime.now().difference(lastShown) <= _kCooldown`
- 제안: `< _kCooldown` (14일 경과 당일 표시 허용)
- 이유: `<=`는 정확히 14일이 지난 순간도 스킵. 의도가 "14일 후부터 재노출"이라면 `<`가 명확. 현재 `<=`도 큰 UX 차이 없으나 문서(테스트케이스)와 코드 의도 일치시킬 것.

**`app_review_service.dart:137-145`** — `_openStoreLink` 예외 처리
- `canLaunchUrl` 실패 시 아무 피드백 없이 종료. 사용자는 버튼 탭 후 아무 반응 없이 끝남.
- 제안: `debugPrint` 외에 향후 개선 가능성 있음. 현재는 허용 가능.

**`app_review_service.dart:150-154`** — `markDismissed()` 미사용
- `[별로예요]` / `[나중에]` 시 `markShown()`이 Step 1 시점에 이미 호출되므로 `markDismissed()`는 호출되지 않음.
- 현재 구현: Step 1 팝업 뜨는 순간 `markShown()` → `lastShown` 기록 → 쿨타임 적용됨.
- `markDismissed()`는 사용되지 않는 dead code. 제거하거나 인라인 주석으로 의도 명시 권장.

**`main_screen.dart:162-166`** — PopScope 뒤로가기 후 `pop` 호출
```dart
onPopInvokedWithResult: (didPop, result) async {
  if (didPop) return;
  await _onWillPop();
  if (mounted) Navigator.of(context).pop(result); // ← 이 pop이 실제로 앱을 종료함
},
```
- `canPop: false`인데 마지막에 `Navigator.pop(result)` 호출 → 팝업 표시 후 앱이 무조건 종료됨.
- 의도가 "팝업 보여주고 항상 종료"라면 OK. "팝업 후 종료 여부를 선택"이 필요하면 수정 필요.
- 현재 UX 시나리오상 팝업 완료 후 종료가 맞으므로 허용 가능. 단 주석으로 명시 권장.

**`trade_review_screen.dart:68-72`** — 리뷰 팝업 후 화면 이동 순서
```dart
if (await reviewService.shouldShow(tradeTriggered: true)) {
  if (mounted) await AppReviewPopup.show(context, reviewService);
}
if (!mounted) return;
context.navigateTo(screen: const MainScreen(), ...);
```
- 팝업이 뜨지 않는 경우에도 `navigateTo` 정상 동작 ✅
- 팝업 완료 후 `mounted` 재확인 ✅ 구현 안전.
