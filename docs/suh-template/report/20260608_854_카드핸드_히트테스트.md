# 카드 핸드에서 본인 물품 선택 시 우측 물품이 선택되는 문제 (히트테스트)

## 개요
홈 탭 카드 핸드에서 본인 물품 카드를 터치했을 때 의도와 다르게 우측 카드가 선택되는 문제를 해결했다. 기존 히트테스트는 카드의 X 중심과 터치점의 X 거리만 비교했고, 검사 순서도 시각적 겹침과 반대였다. 이번 변경으로 검사 순서를 시각적 최상단(index 0)부터로 바로잡고, 카드의 회전각(angle)과 세로 위치(centerY)를 반영해 터치점을 카드 로컬 좌표로 역회전 변환한 뒤 사각형 내부 여부로 판정하도록 정밀화했다.

## 변경 사항
### 카드 핸드 위젯
- `lib/widgets/home_tab_card_hand.dart`: `_findCardAtPosition` 히트테스트 로직 정밀화(+25/-7). 검사 순서를 index 0부터(시각적 최상단)로 변경, 회전각/세로 위치 반영한 역회전 좌표 변환 기반 사각형 판정 도입

## 주요 구현 내용
- 검사 순서 정정: 렌더링이 `_cards.reversed`로 그려져 index 0이 시각적 최상단(Stack 맨 위)이다. 기존에는 역순(`length-1`부터)으로 검사해 사용자가 보는 최상단과 어긋났다. index 0부터 순회해 첫 매치를 반환하도록 바꿔, 보이는 최상단 카드가 선택되게 했다.
- 회전 반영 역변환: 렌더는 `rotateZ(angle)`(tangent + tilt)으로 회전한다. 터치점을 카드 중심 기준 상대 좌표(dx, dy)로 만든 뒤 `-angle`로 역회전(`cosA`, `sinA`)해 카드 로컬 좌표(localX, localY)로 변환한다.
- 사각형 내부 판정: 기존의 `(dx).abs() < _cardWidth/2` X-only 판정을 `localX.abs() < _cardWidth/2 && localY.abs() < _cardHeight/2`로 교체해 회전·세로 위치까지 반영한다.
- 중심 Y 계산: `transform['top']`은 `Positioned.top`과 동일 기준(adjustedTop)이므로 카드 중심 Y는 `top + _cardHeight/2`로 구한다.

## 주의사항
- 전제: hit-test는 `_handlePanStart`(터치 시작) 시점에 호출되며, 이때 카드는 팬/재정렬/pull 애니메이션이 끝난 정적 상태다. 따라서 `_calculateCardTransform`의 정적 목표값(보간 전)으로 판정해도 실제 렌더 위치와 일치한다. 애니메이션 진행 중 hit-test가 필요해지는 변경이 생기면 보간값 기반으로 재검토가 필요하다.
