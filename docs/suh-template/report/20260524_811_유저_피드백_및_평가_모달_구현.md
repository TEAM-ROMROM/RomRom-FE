# 유저 피드백 및 평가 모달 구현

## 개요

앱 스토어 별점/리뷰 유도를 위한 2단계 팝업 시스템을 구현했다. 긍정적 사용자만 필터링해 스토어 리뷰로 유도하고, 부정적 사용자는 쿨타임 처리한다. 백엔드 연동 없이 SharedPreferences 기반 로컬 상태로 관리하며, `in_app_review` 패키지로 iOS/Android 네이티브 리뷰 다이얼로그를 호출한다.

## 변경 사항

### 신규 파일

- `lib/services/app_review_service.dart`: 리뷰 팝업 상태 관리 싱글톤 서비스. SharedPreferences 6개 키로 영구 비활성, 쿨타임, 노출 횟수, 앱 실행 횟수, 첫 실행일, 거래 완료 횟수 관리. `in_app_review` 네이티브 호출 및 스토어 링크 폴백 로직 포함.
- `lib/widgets/common/app_review_popup.dart`: 2단계 CommonModal 시퀀스 제어 위젯. Step 1 감성 질문 → [좋아요] 선택 시 Step 2 리뷰 유도 순서로 진행.

### 기존 파일 수정

- `lib/screens/main_screen.dart`: `initState`에서 `AppReviewService().onAppLaunch()` 호출로 실행 횟수 기록. `PopScope`로 최상위 래핑하여 앱 종료 시도 시 팝업 트리거.
- `lib/screens/trade_review_screen.dart`: `_submit()` 완료 후 `onTradeComplete()` + `shouldShow(tradeTriggered: true)` 체크. 조건 충족 시 `AppReviewPopup.show()` 호출 후 `navigateTo(MainScreen)`.
- `pubspec.yaml`: `in_app_review: ^2.0.9` 추가.

## 주요 구현 내용

### 팝업 플로우

```
[트리거 조건 충족]
        ↓
Step 1: "RomRom을 잘 이용하고 계신가요?"
[좋아요! 😊]         [별로예요 😞]
       ↓                     ↓
Step 2: "스토어 리뷰를     14일 쿨타임
남겨주실래요?"
[리뷰 남기기]        [나중에]
       ↓                     ↓
in_app_review            14일 쿨타임
네이티브 다이얼로그
(실패 시 스토어 링크 폴백)
       ↓
   영구 비활성
```

### 트리거 조건 (2가지)

1. **거래 완료 우선**: 거래 1회 이상 + 앱 사용 3일 이상 → `TradeReviewScreen` 완료 직후
2. **앱 종료 폴백**: 앱 실행 누적 2회 이상 → `MainScreen` `PopScope` 종료 시도 시

### 상태 관리 로직

| 유저 행동 | 상태 변화 |
|----------|----------|
| Step 1 팝업 노출 | `shown_count++`, `last_shown = now` |
| [리뷰 남기기] 클릭 | `disabled = true` (영구 비활성) |
| [별로예요] / [나중에] | 14일 후 재노출 (`last_shown` 이미 기록됨) |
| `shown_count >= 3` | `disabled = true` (영구 비활성) |

## 주의사항

- iOS 네이티브 리뷰 다이얼로그(`SKStoreReviewController`)는 OS가 노출 횟수를 자체 제한하므로 실제로 뜨지 않을 수 있음. 이 경우 스토어 링크 폴백이 동작하지 않으므로 사용자 입장에서 아무 반응 없이 끝날 수 있음 — 의도된 동작.
- `in_app_review`는 디버그 빌드에서 정상 동작하지 않을 수 있음. 실기기 릴리스 빌드로 검증 필요.
- 쿨타임 기준점은 Step 1 팝업이 뜨는 순간(`markShown()`)으로, 사용자 응답 여부와 무관하게 기록됨.
