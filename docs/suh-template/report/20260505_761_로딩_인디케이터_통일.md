## 주요 구현 내용

### CommonLoadingIndicator 설계

```dart
class CommonLoadingIndicator extends StatelessWidget {
  const CommonLoadingIndicator({super.key, this.color = AppColors.primaryYellow, this.size = 24.0});
  const CommonLoadingIndicator.white({super.key, this.size = 18.0}) : color = AppColors.textColorWhite;
}
```

- `color`/`size` 파라미터로 컨텍스트별 커스터마이즈 가능
- `.white` named constructor로 버튼 내부 스피너 간편 사용
- 기존 `CircularProgressIndicator(color: AppColors.primaryYellow, strokeWidth: 2)` 패턴 흡수

### 스켈레톤 패턴

기존 `skeletonizer` 패키지 활용, `AppSkeleton` 래퍼와 동일한 `ShimmerEffect` 설정 통일:
```dart
ShimmerEffect(baseColor: AppColors.opacity10White, highlightColor: AppColors.opacity30White)
```

## 주의사항

- `trade_request_screen.dart`의 버튼 스피너 조건은 `_isSubmitting || _isCheckingExistence`로 main 기준 유지 — 761 원본의 `_isSubmitting`만 체크하던 조건보다 정확함
- `notification_screen.dart`의 `_buildNotificationList`에서 로딩 상태를 skeleton 이전에 별도 분기 처리 — `AppSkeleton` 래퍼와 중복 체크되지 않도록 주의
- `CircularProgressIndicator` 직접 사용 금지 — `CommonLoadingIndicator` 사용 필수 (위젯 주석에 명시됨)
