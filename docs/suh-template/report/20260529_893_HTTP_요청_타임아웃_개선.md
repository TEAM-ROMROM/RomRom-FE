## 🎯 작업 목표

`LoggingHttpClient.defaultTimeout`이 10초로 하드코딩되어 이미지 업로드 등 장시간 요청에 일괄 적용되던 문제를 해결한다. 요청별 timeout 지정 + body read 구간까지 deadline 적용.

---

## 🛠️ 변경 사항

### 1. `lib/utils/log_http_client_interceptor.dart`
- `defaultTimeout` **10초 → 15초** 상향
- 신규 메서드 **`sendWithTimeout(request, timeout)`** 추가 — 요청별 timeout 지정 가능
- 기존 `send(request)`는 `sendWithTimeout(request, defaultTimeout)`로 위임
- **deadline 기반 전 구간 타임아웃**: `send`(헤더 수신) + `stream.toBytes()`(body 읽기) 모두 동일 deadline 사용 → body read 구간 누락 버그 수정
- timeout 초과 시 `TimeoutException` 로그에 단계 표시 (`body read` 등)

### 2. `lib/services/api_client.dart`
- `sendHttpRequest` / `sendMultipartRequest` / `_executeHttpRequest` / `_executeMultipartRequest` 4개 메서드에 **`Duration timeout = const Duration(seconds: 15)`** 파라미터 추가
- 내부 `_client.send(request)` → `_client.sendWithTimeout(request, timeout)`로 교체
- `_executeHttpRequest` GET/POST/PUT/DELETE/PATCH switch 분기 제거 → `http.Request` 통합 처리로 단순화 (29줄 → 20줄)

### 3. `lib/services/apis/image_api.dart`
- `uploadImages` 호출에 **`timeout: const Duration(seconds: 60)`** 명시

### 4. `lib/services/apis/item_api.dart`
- 물품 등록 API에 **`timeout: const Duration(seconds: 60)`** 명시

### 5. `lib/widgets/home_tab_card_hand.dart` (관련 버그 수정)
- `didUpdateWidget`에서 `widget.cards` 변경 감지 시 `_generateCards()` 재실행 누락 수정 — 카드 핸드 상태 미갱신 버그 해결

## 📊 영향 범위

| 항목 | Before | After |
|------|--------|-------|
| 기본 타임아웃 | 10초 (하드코딩) | 15초 (요청별 override 가능) |
| 이미지 업로드 타임아웃 | 10초 → 실패 가능 | 60초 |
| 물품 등록 타임아웃 | 10초 → 실패 가능 | 60초 |
| body read 구간 | 타임아웃 미적용 (무한 대기 가능) | deadline 기반 적용 |

---

## ✅ 핵심 개선 효과

1. **업로드 안정성** — 대용량 이미지/물품 등록 시 timeout 오류 대폭 감소
2. **유연성** — 호출부에서 요청별 timeout 자유롭게 지정
3. **신뢰성** — body read 누락 구간 deadline 적용으로 hang 방지
4. **유지보수성** — `_executeHttpRequest` switch 제거, http.Request 통합 처리로 코드 단순화
