# [버그][요청관리] 보낸요청 카드 갤럭시 프로필 이미지 타원 및 기본 이미지 미표시 수정

## 개요

요청관리 > 보낸요청 탭의 교환 카드에서 발생하던 두 가지 프로필 이미지 버그를 수정했다. 갤럭시(Android)에서 `.w`와 `.h` 스케일 불일치로 프로필 이미지가 타원형으로 표시되던 문제와, 프로필 URL이 없을 때 기본 이미지(`basicProfile.svg`) 대신 `ErrorImagePlaceholder`가 표시되던 문제를 `UserProfileCircularAvatar` 위젯 교체로 동시에 해결했다.

## 변경 사항

### 위젯 수정

- `lib/widgets/sent_request_item_card.dart`:
  - `import 'package:romrom_fe/widgets/user_profile_circular_avatar.dart'` 추가
  - `_buildProfileImage()` 메서드: 기존 `Container + BoxShape.circle + ClipOval + _buildImage()` 조합 제거 → `UserProfileCircularAvatar` 위젯으로 교체
  - `_buildImage()` 메서드: 프로필 이미지 빌더로만 사용되던 메서드 제거 (아이템 이미지는 기존 `CachedImage` 유지)
  - `avatarSize: Size(24.w, 24.w)` — width와 height 모두 `.w` 기준으로 통일하여 갤럭시에서 원형 보장

## 주요 구현 내용

### 버그 1: 타원형 표시 (갤럭시)

기존 코드는 `width: 24.w, height: 24.h`를 사용했는데, `ScreenUtil`에서 `.w`는 화면 너비 기준, `.h`는 화면 높이 기준 스케일이다. 갤럭시는 iOS 대비 화면 비율이 달라 두 값이 달라지고, 결과적으로 원형 컨테이너가 타원이 된다. `Size(24.w, 24.w)`로 통일하여 해결했다. (CLAUDE.md 규칙: 정사각형 요소는 `.w` 기준 사용)

### 버그 2: 기본 프로필 이미지 미표시

기존 `_buildImage()`는 `imageUrl.isEmpty`일 때 `ErrorImagePlaceholder()`를 반환했는데, 이는 에러 UI로 기본 프로필 이미지가 아니다. `UserProfileCircularAvatar`는 `profileUrl`이 `null`이거나 빈 값일 때 내부적으로 `basicProfile.svg`를 자동 폴백하므로, `otherUserProfileUrl.isEmpty ? null : otherUserProfileUrl` 처리로 두 케이스를 모두 커버한다.

## 주의사항

- `UserProfileCircularAvatar`는 `StatefulWidget`이므로 `SentRequestItemCard`가 `StatelessWidget`인 경우에도 Flutter가 내부 상태를 관리하므로 문제없다.
- `hasBorder: true`로 기존과 동일하게 흰색 테두리 유지. `isDeleteAccount: false`로 일반 상태 테두리 색상 사용.
- 아이템 이미지(`_buildItemImage`)의 `CachedImage` + `ErrorImagePlaceholder` 조합은 변경하지 않았다 — 아이템 이미지는 프로필과 다르게 기본 이미지 없이 에러 표시가 적합하다.
