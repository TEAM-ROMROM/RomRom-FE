# 🚀[기능개선][요청관리] 완료 표시 토글 UX 개선

## 개요

요청 관리 화면의 "완료 표시" 토글 동작을 UX 표준에 맞게 수정했다. 기존에는 토글 ON 시 완료된 요청만 단독 표시되어 진행 중인 요청이 사라지는 문제가 있었으며, 토글 라벨도 의미가 불명확했다. 이번 수정으로 토글 ON 시 진행 중 + 완료 항목을 함께 표시하도록 변경하고, 라벨을 '완료 포함'으로 명확히 했다.

## 변경 사항

### 필터링 로직 수정
- `lib/screens/request_management_tab_screen.dart`: `_showCompletedRequests` 토글 ON 조건을 `TRADED` 상태만 반환 → `true`(전체 반환)로 변경

### 라벨 텍스트 수정
- `lib/screens/request_management_tab_screen.dart`: 토글 옆 텍스트 `'완료 표시'` → `'완료 포함'`으로 변경

### 빈 상태 메시지 단일화
- `lib/screens/request_management_tab_screen.dart`: 토글 상태에 따라 분기되던 빈 상태 메시지를 `'진행 중인 요청이 없습니다'`로 통일

## 주요 구현 내용

토글 OFF(기본): `TRADED` 상태 제외 → 진행 중인 요청만 표시  
토글 ON: 필터 없이 전체 반환 → 진행 중 + 완료 항목 혼합 표시

당근마켓 등 일반적인 거래 앱에서 채택하는 "완료 포함해서 보기" 패턴과 동일하게 맞춤.

## 주의사항

- 보낸 요청 탭에는 동일한 `_showCompletedRequests` 상태를 공유하므로 받은 요청/보낸 요청 양쪽 모두에 동일하게 적용됨
- 완료 항목의 시각적 구분(예: 흐리게 처리)은 이번 범위에서 제외 — 추후 개선 가능
