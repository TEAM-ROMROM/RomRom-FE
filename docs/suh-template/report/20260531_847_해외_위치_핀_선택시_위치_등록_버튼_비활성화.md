## 🎯 작업 목표

물품 등록 위치 선택 화면에서 해외 좌표에 핀을 찍으면 위치 등록 버튼이 영구 비활성화되고, 추가로 stale state로 인해 직전 국내 주소가 그대로 등록될 수 있는 silent data corruption 문제를 함께 해결한다.

---

## 🔍 근본 원인 (Root Cause)

`LocationService.getAddressFromCoordinates()`는 네이버 Reverse Geocoding API를 호출한다. 네이버 API는 **국내 좌표만 지원**하므로 해외 좌표는 `results=[]`를 반환하고 함수는 `null`을 돌려준다.

`ItemRegisterLocationScreen._updateAddress()`는 기존에 `address != null`일 때만 `setState`를 수행했다. 그 결과:

1. **해외 진입 시 영구 비활성** — 첫 좌표가 해외면 `_selectedAddress`가 계속 null → 버튼 disabled
2. **국내 → 해외 이동 시 stale state** — `_selectedAddress`가 직전 국내 주소를 그대로 유지 → 버튼은 활성, 그러나 등록되는 주소가 핀 위치와 무관한 잘못된 국내 주소가 됨 (이슈에는 명시되지 않은 더 심각한 부수 버그)

---

## 🛠️ 변경 사항

### `lib/screens/item_register_location_screen.dart`

#### 1. `_updateAddress` — null 응답도 명시적 반영
```dart
Future<void> _updateAddress(NLatLng position) async {
  final address = await _locationService.getAddressFromCoordinates(position);
  if (!mounted) return;
  // 해외 좌표 등으로 주소를 받지 못한 경우 _selectedAddress=null 로 리셋해
  // 직전 국내 주소가 남아 잘못된 위치가 등록되는 stale state 버그를 막는다.
  setState(() {
    _selectedPosition = position;
    _selectedAddress = address;
  });
}
```
- null도 setState로 반영 → 버튼 상태가 항상 현재 핀 좌표와 일치
- `_selectedPosition`은 주소 성공/실패와 무관하게 항상 갱신

#### 2. 안내 배너 UI 추가
```dart
if (_currentPosition != null && _selectedAddress == null)
  Positioned(
    left: 24.w, right: 24.w, bottom: 110.h,
    child: Container(
      padding: const EdgeInsets.symmetric(horizontal: 16, vertical: 10),
      decoration: BoxDecoration(
        color: AppColors.opacity80Black,
        borderRadius: BorderRadius.circular(8),
      ),
      child: Text(
        '국내 위치만 등록할 수 있어요',
        textAlign: TextAlign.center,
        style: CustomTextStyles.p2.copyWith(color: AppColors.textColorWhite),
      ),
    ),
  ),
```
- 주소 미수신 상태에서 사용자에게 원인을 명확히 안내
- `_selectedAddress == null` 조건으로 자동 노출/숨김 (해외 ↔ 국내 이동에 즉시 반응)

#### 3. import
- `lib/models/app_theme.dart` 추가 (`CustomTextStyles` 사용)
