📝 현재 문제점
---

- 외부 딥링크(`https://romrom-c4008.web.app/item?itemId=XXX`)를 공유받은 사용자가 앱 미설치 상태일 경우, Firebase Hosting 기본 페이지(`https://romrom-c4008.web.app/`)로 이동됨
- 현재 해당 페이지는 앱 다운로드 안내 기능이 없거나 브랜딩이 부재한 빈 페이지임
- 앱 미설치 사용자가 딥링크를 통해 유입될 경우 서비스에 대한 인상을 남기지 못하고 이탈하게 됨

🛠️ 해결 방안 / 제안 기능
---

- `https://romrom-c4008.web.app/` (Firebase Hosting) 페이지를 **RomRom 앱 소개 랜딩 페이지**로 교체
- 랜딩 페이지에 포함할 내용:
  - 앱 브랜딩 (로고, 슬로건, 앱 설명)
  - iOS App Store / Google Play Store 다운로드 버튼
  - 딥링크 파라미터(`itemId` 등)를 보존하여, 앱 설치 후 해당 콘텐츠로 바로 이동 가능하도록 처리 (Deferred Deep Link 방식)
- 관련 파일:
  - [`lib/models/app_urls.dart`](lib/models/app_urls.dart) — `itemShareBaseUrl = "https://romrom-c4008.web.app"` 정의
  - [`lib/utils/deep_link_router.dart`](lib/utils/deep_link_router.dart) — `_hostingDomain = 'romrom-c4008.web.app'` 기반 라우팅 처리

⚙️ 작업 내용
---

- [ ] Firebase Hosting 프로젝트에 앱 소개 랜딩 페이지(HTML/CSS/JS) 작성
- [ ] iOS / Android 스토어 다운로드 버튼 추가 (`lib/models/app_urls.dart`의 `androidStoreUrl`, `iosStoreUrl` 활용)
- [ ] 딥링크 파라미터 보존 처리 (URL 파라미터를 로컬스토리지 또는 세션에 저장 후 앱 설치 유도)
- [ ] 페이지 배포 및 기존 `https://romrom-c4008.web.app/` 경로에 반영
