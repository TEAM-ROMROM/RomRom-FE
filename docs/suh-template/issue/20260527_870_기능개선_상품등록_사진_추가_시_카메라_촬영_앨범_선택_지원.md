# 🚀[기능개선][상품등록] 사진 추가 시 카메라 촬영/앨범 선택 지원

- 라벨: 작업전
- 담당자: Cassiiopeia

---

📝 현재 문제점
---

- 상품 등록 화면에서 사진 칸을 누르면 **곧바로 로컬 갤러리 다중선택**으로만 진입합니다.
- 사용자가 그 자리에서 **바로 사진을 촬영해 등록하는 경로**가 없습니다.
- 대부분의 거래/커머스 앱은 사진 추가 시 "촬영"과 "앨범에서 선택"을 고를 수 있는데, 현재는 앨범 선택만 가능해 사용성이 떨어집니다.

🛠️ 해결 방안 / 제안 기능
---

- 사진 칸을 탭하면 **바텀시트**가 올라와 두 가지 중 선택하도록 개선합니다.
  - 📷 사진 촬영하기 (카메라)
  - 🖼️ 앨범에서 선택하기 (기존 갤러리 다중선택)
- **카메라 촬영**: 1장 촬영 후 별도 크롭/편집 없이 기존 이미지 목록에 바로 추가합니다.
- **앨범 선택**: 기존 다중선택 흐름을 그대로 유지합니다.
- **권한 처리** (기업 표준 방식):
  - 카메라 권한을 사전 체크하고, 거부 시 안내 스낵바를 노출합니다.
  - 영구 거부 상태면 "설정으로 이동" 안내를 제공합니다.
- 이미지 최대 10장 한도, 중복 요청 방지 등 기존 제약은 그대로 적용합니다.

⚙️ 작업 내용
---

- 공통 소스 선택 바텀시트 위젯 추가 (`lib/widgets/common/`)
- 카메라 권한 확보 헬퍼 추가 (`permission_handler` 사용)
- 상품 등록 폼(`lib/widgets/register_input_form.dart`)의 사진 추가 로직에 소스 선택 분기 적용
- iOS `Info.plist` 카메라 사용 설명(`NSCameraUsageDescription`) 추가
- Android `AndroidManifest.xml` 카메라 권한 추가
- 적용 범위: **상품 등록 화면만** (채팅/프로필 이미지 변경은 이번 범위 제외)
