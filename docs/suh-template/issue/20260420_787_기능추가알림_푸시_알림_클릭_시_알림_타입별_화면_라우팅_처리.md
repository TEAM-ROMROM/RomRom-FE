# ⚙️[기능추가][알림] 푸시 알림 클릭 시 알림 타입별 화면 라우팅 처리

- 라벨: 작업 전
- 담당자: -

---

📝 현재 문제점
---

- 푸시 알림(FCM)을 수신한 뒤 해당 알림을 탭해도 앱이 단순히 열릴 뿐, 알림 종류에 맞는 화면으로 이동하지 않음
- `FirebaseMessaging.onMessageOpenedApp` 리스너는 등록되어 있으나 (`lib/services/firebase_service.dart:72`) 실제 라우팅 로직이 없음
- `getInitialMessage()` 처리가 누락되어 있어, 앱이 종료된 상태에서 알림을 탭해 실행될 경우 초기 라우팅이 동작하지 않음
- 알림 타입(`NotificationType`)은 이미 정의되어 있으나 (`lib/enums/notification_type.dart`) 탭 시 라우팅에 활용되지 않음

🛠️ 해결 방안 / 제안 기능
---

- `FirebaseMessaging.onMessageOpenedApp` 콜백에서 메시지 데이터의 알림 타입을 파싱하고 `DeepLinkRouter`로 라우팅 연결
- `main.dart` 또는 앱 초기화 시점에 `getInitialMessage()` 처리 추가 (앱 종료 상태에서 알림 탭으로 실행된 경우 대응)
- 알림 타입별 라우팅 규칙:
  - `CHAT_MESSAGE_RECEIVED` → `chatRoomId` 파라미터로 ChatRoomScreen 이동 (`romrom://chat/room?chatRoomId=XXX`, `lib/utils/deep_link_router.dart:87`)
  - `TRADE_REQUEST_RECEIVED` → ItemDetailDescriptionScreen 이동 (`romrom://item/detail?itemId=XXX`)
  - `ITEM_LIKED` → 해당 아이템 상세 이동
  - `SYSTEM_NOTICE` → 알림 목록 또는 공지 화면 이동
- iOS / Android 모두 동일 로직으로 처리 (플랫폼 분기 불필요, FCM 페이로드 기반)

⚙️ 작업 내용
---

- `lib/services/firebase_service.dart`: `onMessageOpenedApp` 콜백에 알림 타입 파싱 및 `DeepLinkRouter.openFromUri()` 호출 로직 추가
- `lib/main.dart`: `getInitialMessage()` 처리 블록 추가 (앱 콜드 스타트 시)
- `lib/utils/deep_link_router.dart`: 채팅 알림 라우팅 경로 검증 (`romrom://chat/room?chatRoomId=XXX`)
- 백엔드에서 FCM 페이로드에 `notificationType`, `chatRoomId`, `itemId` 등 라우팅에 필요한 데이터 포함 여부 확인 필요 (BE 이슈 별도 등록)
