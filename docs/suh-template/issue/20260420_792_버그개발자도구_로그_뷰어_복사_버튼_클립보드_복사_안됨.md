🗒️ 설명
---

개발자 도구(Debug Log Panel)에서 로그 복사 버튼 클릭 시 클립보드에 복사가 되지 않는 문제가 있다.

현재 코드 분석:

1. **`await` 누락**: `lib/debug/widgets/debug_log_panel.dart:106`의 `_copyLogs()`에서 `Clipboard.setData()`를 `await` 없이 호출 중. 비동기 작업이 완료되기 전에 "복사됨!" 피드백이 표시되어 실제 복사 성공 여부와 무관하게 UI가 갱신됨.

```dart
// 현재 (문제)
void _copyLogs() {
  Clipboard.setData(ClipboardData(text: text)); // await 없음
  setState(() => _showCopiedFeedback = true);   // 복사 완료 여부 미확인
}
```

2. **1000줄 대용량 텍스트 문제**: `LogCapture._maxBufferSize = 1000` 이므로 최대 1000개 로그가 누적될 수 있음. iOS/Android의 클립보드 플랫폼 채널이 대용량 문자열 전달 시 실패할 가능성 있음.

🔄 재현 방법
---

1. 앱 실행 후 개발자 도구 플로팅 버튼 활성화
2. 로그 뷰어 패널 열기
3. 로그가 1000건 가까이 쌓인 상태에서 복사 버튼 클릭
4. 다른 앱에 붙여넣기 시도 → 내용이 없거나 이전 클립보드 내용이 유지됨

📸 참고 자료
---

- 관련 파일:
  - `lib/debug/widgets/debug_log_panel.dart:96~111` — `_copyLogs()` 복사 로직
  - `lib/debug/log_capture.dart:16` — `_maxBufferSize = 1000` 버퍼 크기 설정

✅ 예상 동작
---

- 복사 버튼 클릭 시 현재 필터된 로그 전체가 클립보드에 정상 복사되어야 한다.
- 복사 실패 시 "복사됨!" 대신 오류 피드백이 표시되어야 한다.
