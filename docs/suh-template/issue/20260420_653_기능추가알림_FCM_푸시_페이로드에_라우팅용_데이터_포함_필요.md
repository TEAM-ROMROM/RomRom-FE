# ⚙️[기능추가][알림] FCM 푸시 페이로드에 라우팅용 데이터 포함 필요

- 라벨: 작업 전
- 담당자: -

---

📝 현재 문제점
---

- 푸시 알림을 수신하고 탭했을 때 클라이언트(iOS/Android)가 알림 타입에 맞는 화면으로 이동하려면, FCM 페이로드의 `data` 필드에 라우팅에 필요한 정보가 포함되어 있어야 함
- 현재 FCM 발송 시 `data` 필드에 `notificationType`, `chatRoomId`, `itemId` 등 클라이언트 라우팅에 필요한 값이 포함되는지 확인 필요
- 값이 누락된 경우 프론트엔드에서 알림 탭 시 적절한 화면으로 이동할 수 없음

🛠️ 해결 방안 / 제안 기능
---

- FCM 발송 시 `data` 필드에 아래 라우팅 데이터를 알림 타입에 맞게 포함:
  - 공통: `notificationType` (예: `CHAT_MESSAGE_RECEIVED`, `TRADE_REQUEST_RECEIVED`, `ITEM_LIKED`, `SYSTEM_NOTICE`)
  - `CHAT_MESSAGE_RECEIVED`: `chatRoomId`
  - `TRADE_REQUEST_RECEIVED`: `itemId`, `tradeRequestHistoryId`
  - `ITEM_LIKED`: `itemId`
  - `SYSTEM_NOTICE`: (라우팅 대상 없음, 생략 가능)
- `notification` 필드 대신 `data-only` 메시지로 발송하는 방식 권장 (iOS 백그라운드에서도 `onMessage` 핸들러로 수신 가능)

⚙️ 작업 내용
---

- FCM 발송 서비스에서 알림 타입별 `data` 필드 구성 로직 추가/수정
- 페이로드 예시:
  ```json
  {
    "data": {
      "notificationType": "CHAT_MESSAGE_RECEIVED",
      "chatRoomId": "12345"
    }
  }
  ```
- 프론트엔드 연동 확인 필요 (FE 이슈 별도 등록)
