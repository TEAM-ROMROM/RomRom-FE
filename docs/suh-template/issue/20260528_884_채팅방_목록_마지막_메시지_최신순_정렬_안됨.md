# ❗[버그][채팅] 채팅방 목록이 마지막 메시지 최신순으로 정렬되지 않음

라벨: 작업전
담당자: Cassiiopeia

🗒️ 설명
---

- 채팅 탭의 채팅방 목록이 **마지막 메시지 시각(`lastMessageTime`) 최신순**으로 정렬되어야 하는데, 그렇지 않다.
- 새 메시지가 도착해도 해당 채팅방이 항상 맨 위로 올라오지 않는다.
- 앱 진입 / 새로고침 / WebSocket 재연결 시 정렬이 깨진다.

원인:
- 백엔드 `getRooms`가 채팅방 생성일(`createdDate`) 내림차순으로 정렬한다. 마지막 메시지 시각 기준이 아니라 채팅방이 만들어진 순서로 고정된다.
- 프론트는 WebSocket 메시지 수신 시점에만 해당 방을 리스트 맨 앞으로 옮긴다(`insert(0)`). 따라서 WebSocket 미수신 경로(초기 로드, 재연결 직후 재조회 등)에서는 최신순이 보장되지 않는다.
- `ChatRoom`(PostgreSQL)과 `ChatMessage`(MongoDB)가 분리되어 있어 백엔드에서 `lastMessageTime` 기준 SQL 정렬이 불가능하다. 정렬을 프론트에서 처리한다.

🔄 재현 방법
---

1. 채팅 탭 진입 (채팅방 2개 이상 존재)
2. 가장 오래전에 생성된 채팅방에 새 메시지 수신
3. 해당 채팅방이 목록 최상단으로 정렬되는지 확인 → 최신순 정렬이 유지되지 않음

📸 참고 자료
---

- 관련 파일: `lib/screens/chat_tab_screen.dart`
  - 페이지 누적: `_loadChatRooms` (`_chatRoomsDetail.addAll(...)`)
  - WebSocket 수신: `_onMessageReceived` (`removeAt` + `insert(0)`)

✅ 예상 동작
---

- 채팅방 목록이 항상 마지막 메시지 시각 기준 내림차순(최신 메시지가 맨 위)으로 표시된다.
- 새 메시지 수신 / 초기 로드 / 새로고침 / 재연결 등 리스트가 갱신되는 모든 경로에서 정렬이 일관되게 유지된다.
- 메시지가 없는 채팅방은 백엔드가 `createdDate`로 채워주므로 생성순으로 자연스럽게 배치된다.
