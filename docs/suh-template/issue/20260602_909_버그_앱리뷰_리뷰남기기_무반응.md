🗒️ 설명
---

앱 리뷰 유도 팝업의 2단계(Step 2) "스토어에 리뷰를 남겨주실래요?"에서 **[리뷰 남기기]** 버튼을 눌러도 인앱 리뷰 카드도, 스토어 화면도 뜨지 않고 그대로 닫히는 무반응 현상이 발생합니다.

- 테스트 초기에는 몇 번 정상적으로 인앱 리뷰 카드가 떴으나, 어느 시점 이후부터 [리뷰 남기기]를 눌러도 아무 화면도 나타나지 않습니다.
- 사용자 입장에서는 "리뷰를 남기겠다"고 명시적으로 버튼을 눌렀는데 아무 일도 일어나지 않아 동작이 깨진 것처럼 보입니다.
- Android·iOS 공통으로 재현됩니다.

🔄 재현 방법
---

1. 앱 리뷰 유도 팝업 노출 (거래 완료/앱 실행 트리거 충족 시)
2. Step 1 "잘 이용하고 계신가요?" → **[좋아요!]** 선택
3. Step 2 "스토어에 리뷰를 남겨주실래요?" → **[리뷰 남기기]** 선택
4. 인앱 리뷰 카드도 스토어도 뜨지 않고 팝업이 그냥 닫힘 (특히 짧은 기간 내 재시도하거나 이미 한 번 리뷰 흐름을 탄 기기/계정에서 재현)

📸 참고 자료
---

- 관련 코드: `lib/services/app_review_service.dart`
- 원인 분석:
  - `[리뷰 남기기]` 클릭 시 `requestReviewAndDisable()` → `_requestReview()` 경로로 in_app_review 패키지의 `requestReview()`(구글/애플 인앱 리뷰 API)를 호출합니다.
  - `requestReview()`는 **카드 표출 여부·성공 여부를 반환하지 않습니다**(OS·플러그인 정책). 인앱 리뷰는 시스템이 표출 여부를 결정하며, 할당량(quota) 제한이나 시스템 거절 시 에러 없이 아무 동작도 하지 않고 끝납니다(silent no-op).
  - 기존 로직은 `isAvailable()`이 `true`이면 `requestReview()` 호출 후 무조건 `return` 하여, 카드가 실제로 안 떠도 스토어 폴백(`_openStoreLink`)에 도달하지 못합니다. 폴백은 `isAvailable()`이 `false`인 기기(플레이 서비스 없는 환경 등)에서만 동작합니다.
  - 결과적으로 "카드가 안 떴는데 스토어도 안 열리는" 무반응 상태가 됩니다.
- 참고: 테스트/내부 배포 환경, 이미 리뷰한 기기·계정, 짧은 기간 반복 호출 시 인앱 리뷰 카드가 표출되지 않는 것은 시스템 정책상 정상 동작이며, 이것이 버그처럼 보이는 무반응의 원인입니다.

✅ 예상 동작
---

- 사용자가 **[리뷰 남기기]** 를 명시적으로 눌렀을 때는, 인앱 리뷰 카드가 뜨지 않는 경우라도 **스토어 리뷰 페이지로 이동**하여 리뷰 작성 경로가 끊기지 않아야 합니다.
- 인앱 리뷰 카드가 정상 표출되는 경우에는 스토어로 중복 이동하지 않아야 합니다.
- Android·iOS 모두 동일하게 동작해야 합니다.

> 보완 방향: 인앱 리뷰 호출 후 앱 라이프사이클(inactive/paused 전환 여부)을 짧은 시간 감시하여 카드 표출 여부를 추정하고, 표출되지 않았다고 추정되면 `openStoreListing()`으로 스토어 리뷰 페이지로 폴백합니다. (라이프사이클 기반 추정이므로 100% 정확하지는 않은 휴리스틱이며, 특히 iOS에서 한계가 있을 수 있습니다.)
