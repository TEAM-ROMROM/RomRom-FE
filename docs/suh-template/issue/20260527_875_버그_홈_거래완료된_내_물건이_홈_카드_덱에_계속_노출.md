# ❗[버그][홈] 거래완료된 내 물건이 홈 카드 덱에 계속 노출

라벨: 작업전
담당자: Cassiiopeia

---

🗒️ 설명
---

거래가 완료(EXCHANGED)되어도 홈 화면 하단 카드 덱(`HomeTabCardHand`)에 해당 물건이 계속 표시됩니다.

- 거래완료 후에도 내 물건 카드가 홈 카드 덱에서 사라지지 않음
- 홈 화면에 새로고침 기능이 없어, 사용자가 **앱을 껐다 켜야** 카드가 사라짐
- 실제 사용자 제보: "제 거 거래완료 되면 카드 하나 빠져야 하는데 그대로 노출되어서 어플을 껐다 켰습니다"

**BE는 정상**입니다. 서버 로그(`romrom (1).log`, 2026-05-27 14:51:17.884 `ChatTradeCompletionService`)에서 거래완료 시 item이 정상적으로 `EXCHANGED`로 변경되는 것을 확인했습니다. FE 상태관리 누락 문제입니다.

🔄 재현 방법
---

1. 내 물건을 등록하고 거래 요청을 받아 채팅방에서 거래를 진행
2. 채팅방에서 거래완료를 확정 (또는 상대방이 거래완료를 확정)
3. 거래 후기 작성을 건너뛰거나, 상대방이 완료한 경우
4. 홈 화면으로 돌아오면 거래완료된 내 물건이 하단 카드 덱에 그대로 남아있음

📸 참고 자료
---

원인 분석 (FE):

- `getMyItems`는 `itemStatus: AVAILABLE` 필터로 조회함 (`lib/services/apis/item_api.dart:155`) → **재조회만 하면 EXCHANGED 물건은 응답에서 자동 제외됨**
- 그러나 `_loadMyCards()`가 `initState`에서 **1회만** 호출됨 (`lib/screens/home_tab_screen.dart:96`)
- `MainScreen`이 `IndexedStack` 사용 (`lib/screens/main_screen.dart:133`) → 탭 전환해도 홈 탭 state가 메모리에 유지되어 stale한 `_myCards`가 남음
- 거래완료 후 내 카드 목록 재조회를 트리거하는 경로가 없음. 후기 작성 완료 후 `pushAndRemoveUntil(MainScreen)`으로 새 `MainScreen`이 생성될 때만(`lib/screens/trade_review_screen.dart:59`) 우연히 갱신됨 → 후기 스킵 / 상대방이 완료 / 앱 미종료 시 카드가 빠지지 않음

거래완료 발생 지점 (둘 다 `lib/screens/chat_room_screen.dart`):

- 확정자: `_onConfirmTradeRequest()` → `confirmTradeCompletion()` 성공 (`chat_room_screen.dart:715`)
- 요청자(상대방): WebSocket `MessageType.tradeCompleted` 수신 (`chat_room_screen.dart:288`)

✅ 예상 동작
---

- 거래완료 즉시 홈 화면 카드 덱에서 해당 물건이 사라져야 함
- 채팅방에서 거래완료를 확정하거나, 상대방이 확정한 경우(WebSocket 수신) 모두 홈 카드 덱이 갱신되어야 함
- 거래완료로 내 물건이 0개가 되면 홈 블러 / 등록 안내가 다시 표시되어야 함
- 앱을 껐다 켜지 않아도 즉시 반영되어야 함

해결 방향:

- 단일 전역 이벤트 버스(`AppEventBus`) + 타입 기반 이벤트(`TradeCompletedEvent`) 신규 도입
- 거래완료 2개 지점(확정자 / 요청자 WebSocket 수신)에서 `emitTradeCompleted()` 발행
- 홈 탭이 해당 이벤트를 구독해 `_loadMyCards()` 재호출 (거래완료 물건은 `AVAILABLE` 필터에서 자동 제외)
- 설계 문서: `docs/superpowers/specs/2026-05-27-trade-completion-home-card-refresh-design.md`
