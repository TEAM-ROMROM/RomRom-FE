description: 보낸요청 카드에서 상대방 프로필 이미지가 없을 때 basicProfile.svg 대신 ErrorImagePlaceholder가 표시되는 버그
type: bug
---

🗒️ 설명
---

- 요청관리 > 보낸 요청 탭의 교환 카드에서 상대방 프로필 URL이 없거나 빈 값일 때 기본 프로필 이미지(`basicProfile.svg`)가 표시되어야 하나, 현재 `ErrorImagePlaceholder()`가 표시됨
- `_buildImage()`에서 `imageUrl.isEmpty` 체크 시 `ErrorImagePlaceholder()`를 반환하는데, 이는 에러 상황 표시 위젯으로 기본 프로필 이미지 역할을 하지 않음
- `UserProfileCircularAvatar` 위젯은 이미 `basicProfile.svg` 폴백 처리를 내장하고 있으나, 해당 카드에서 사용되지 않음

🔄 재현 방법
---

1. 상대방 프로필 이미지 URL이 없는 교환 요청 카드 확인
2. 요청관리 > 보낸 요청 탭 이동
3. 해당 카드 오른쪽 하단 프로필 영역 확인 → 기본 프로필 이미지 미표시

📸 참고 자료
---

- `lib/widgets/sent_request_item_card.dart` — `_buildImage()` 메서드 (260~266줄)
- `lib/widgets/user_profile_circular_avatar.dart` — 기본 프로필 폴백 내장 위젯
- `assets/images/basicProfile.svg` — 기본 프로필 이미지 에셋

✅ 예상 동작
---

- 프로필 URL이 없을 때 `basicProfile.svg` 기본 이미지가 표시되어야 함
- `UserProfileCircularAvatar` 위젯으로 교체하면 자동으로 폴백 처리됨
