📝 현재 문제점
---

- 홈 피드 / 마이페이지 좋아요 목록의 좋아요 버튼을 누르면 **API 응답이 도착할 때까지 UI 변화가 없음**
- 사용자가 버튼을 눌렀는데 즉각 반응이 없으니 "눌린 건가?" 의심하게 되고, 그 사이 한 번 더 누르거나 다른 동작 시도
- 정확성(서버 응답 동기화)을 우선하느라 UX 측면의 즉시성이 희생됨
- 물품 상세 화면(`item_detail_description_screen.dart`)은 이미 Optimistic Update가 적용되어 있어 다른 화면과 동작이 불일치

🛠️ 해결 방안 / 제안 기능
---

- **Optimistic Update 패턴**으로 통일: 버튼을 누른 즉시 UI(좋아요 상태 + 카운트) 변경, 그 다음 백그라운드로 API 호출
- API 성공 시: 서버 응답 값으로 보정 (서버-클라 불일치 대비)
- API 실패 시: 변경 직전 값으로 롤백 + 에러 안내
- 기존에 도입된 패턴(`item_detail_description_screen.dart` 1030~1054 라인)을 다른 좋아요 버튼에도 동일하게 적용

⚙️ 작업 내용
---

- `lib/widgets/home_feed_item_widget.dart` 286~306 라인 좋아요 핸들러를 Optimistic 방식으로 변경
- `lib/screens/my_page/my_like_list_screen.dart` 좋아요 취소 핸들러(약 330라인)를 Optimistic 방식으로 변경
- 기존 `_isLiking` 중복 클릭 방지 로직은 유지 (Optimistic 적용 후에도 in-flight 동안의 추가 클릭은 무시)
- 실패 케이스 롤백 시 `CommonSnackBar`로 사용자 안내 추가 검토
